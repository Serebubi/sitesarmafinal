# HANDOFF.md

## Короткая передача состояния

Текущий фокус проекта: довести публичные и пользовательские сценарии `Сарма Экспресс` / `SUPERBOX` до целостного состояния, где важные страницы уже не только стилизованы, но и реально работают.

Последние важные изменения уже внесены в код и выгружены в репозиторий `Serebubi/sarmaexpress` в ветку `main`.

## Самое важное

- Главная страница - `Сарма Экспресс`.
- `SUPERBOX` остаётся на `/superbox`, но ключевые flow уже стилизованы под Sarma.
- Калькулятор доставки уже считает реальные суммы по тарифной сетке.
- Отслеживание теперь работает только по номеру заказа.

## Последние важные изменения

1. Реализован рабочий калькулятор доставки на `/calculator`.
2. Вынесена тарифная модель в `frontend/lib/delivery-tariffs.ts`.
3. В калькуляторе добавлены:
   - города,
   - коэффициенты маршрутов,
   - сроки,
   - грузчики,
   - этажность,
   - лифт,
   - частный дом,
   - спецтариф `500+ кг`.
4. В `order_lookup` убран поиск по телефону.
5. Обновлены тексты и ошибки отслеживания: теперь там только номер заказа.
6. Обновлены служебные контекстные файлы (`AGENT.md`, `CURRENT_STATUS.md`, `PROJECT_CONTEXT.md`, `HANDOFF.md`).

## Что проверять при следующей задаче

1. `git status --short --branch`
2. `git remote -v`
3. При задаче по UI:
   - `npm run build -w frontend`
4. При задаче по калькулятору:
   - проверить `delivery-calculator-page.tsx`
   - проверить `delivery-tariffs.ts`
5. При задаче по отслеживанию:
   - проверить `superbox-app.tsx`

## Возможные следующие задачи

- довести оставшиеся success-экраны Superbox до Sarma-стиля;
- расширить/уточнить логику калькулятора, если пользователь даст новую редакцию Excel или формул;
- доработать отдельные flow интернет-магазинов;
- синхронизировать backend, если пользователь попросит завязать UI на реальные данные глубже.

## Осторожно

- не коммитить `.serena/` без необходимости;
- не коммитить локальный Excel `Тарифы Сарма Экспресс.xlsx`, если пользователь отдельно этого не просил;
- не откатывать чужие локальные изменения;
- не трогать backend/Bitrix для чисто визуальных задач.
