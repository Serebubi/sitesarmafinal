## Что сделано

1. В `pickup_paid` настроены анкеты маркетплейсов по `Формы_фух.xlsx`:
   - Wildberries
   - Ozon
   - Яндекс Маркет
   - Lamoda
   - Авито
   - 5Post
   - DPD
   - Детский Мир
   - Золотое Яблоко
   - Лэтуаль
   - СДЭК

2. Для всех этих анкет сделан единый UX:
   - сразу после `Выбран маркетплейс` идёт выбор пункта выдачи;
   - в выпадающем списке только 8 пунктов из Excel;
   - ниже выбора появляется адрес, куда заказывать в Ростове-на-Дону;
   - кнопка отправки требует согласия с условиями.

3. `Заказ по ссылке` (`pickup_standard`) обновлён:
   - доступны только Wildberries, Ozon, Яндекс Маркет;
   - добавлен выбор пункта выдачи и блок адреса, куда заказывать;
   - добавлена дополнительная информация;
   - условия: комиссия за выкуп 15%, возврат бесплатно.

4. Добавлена отмена заказа:
   - новый маршрут `/cancel-order`;
   - пункт `Отмена заказа` в меню `Услуги`;
   - flow с вводом номера, проверкой, подтверждением и переводом заказа в `CANCELLED`.

5. Добавлен дисклеймер:
   - блок внизу страницы выбора маркетплейсов;
   - ссылка `Дисклеймер` в анкетах;
   - модальное микро-окно на открытой странице.

6. Обновлены shared/backend API:
   - `additionalInfo` для `pickup_standard`;
   - правила обязательности полей для маркетплейсов;
   - сохранение и передача новых полей в Bitrix.

## Что важно для следующего агента

- Не добавлять `Формы_фух.xlsx` в Git без отдельного запроса.
- При новой правке маркетплейса сначала сверять соответствующий лист Excel.
- Если меняются поля формы, обновлять UI, `frontend/lib/api.ts`, `shared/src/index.ts`, backend route, storage и Bitrix вместе.
- Пользователь будет проверять анкеты на соответствие Excel; возможны точечные корректировки по текстам и адресам.

## Не коммитить без просьбы

- локальные Excel-файлы;
- PNG/скриншоты из корня;
- `frontend-start-*.log`;
- `node_modules`;
- `.next`;
- `storage/data/orders.json`, если пользователь отдельно не попросит сохранить тестовые заказы.
