## Что завершено

### Главная и навигация

- `/` рендерит `SarmaExpressPage`.
- Верхняя плашка вынесена в `frontend/components/sarma-express-header.tsx`.
- Навигация разделена корректно:
  - `Калькулятор` -> `/calculator`
  - `Отслеживание` -> `/superbox?flow=order_lookup`
  - `Бизнесу` -> `/superbox?flow=business`
  - `Тарифы` -> `/superbox?flow=tariffs`
  - `Доставка из интернет-магазинов` -> `/superbox?flow=pickup_paid`
  - `Отправления в РФ` -> `/superbox?flow=ship_russia`
  - `Пункты выдачи` -> `/pickup-points`
- На главной все 4 карточки `Наши услуги` ведут на `/superbox?flow=pickup_paid`.

### Калькулятор

Файлы:

- `frontend/components/delivery-calculator-page.tsx`
- `frontend/lib/delivery-tariffs.ts`

Сделано:

- калькулятор считает по тарифной сетке из Excel;
- поддержаны города, коэффициенты маршрутов и сроки;
- считается расчётный вес `max(факт, объём * 200)`;
- учитываются:
  - базовый тариф по диапазону веса;
  - коэффициент направления;
  - курьерский фикс;
  - грузчики;
  - подъём без лифта;
  - провоз в лифте;
  - занос в частный дом свыше 25 м;
- для отправлений от `500 кг` показывается спецтариф вместо обычного расчёта.

Контрольный сценарий, уже проверенный в браузере:

- `Ростов-на-Дону -> Донецк`
- вес `20`
- объём `0.12`
- этаж `5`
- лифта нет
- грузчики нужны
- результат: `3 150 ₽`

## Что осталось / риски

- Финальные success-экраны некоторых flow ещё могут оставаться не полностью приведёнными к Sarma-стилю.
- Калькулятор использует логическую модель из Excel, но сам Excel не закоммичен в репозиторий; исходник лежит локально у пользователя.
- Backend-логика и Bitrix специально почти не трогались.
- Яндекс.Карты локально могут ругаться на `Invalid API key`, если невалиден `NEXT_PUBLIC_YANDEX_MAPS_API_KEY`.
