# PROJECT_CONTEXT.md

## Общий контекст

Проект состоит из:

1. `Сарма Экспресс` - публичная витрина, калькулятор, страницы услуг и карта пунктов выдачи.
2. `SUPERBOX` - приложение для заказов интернет-магазинов, отслеживания, тарифов, доставки по РФ, заказа по ссылке, отмены заказов и бизнес-сценариев.

Основной стиль: светло-синий фон, белые/светлые панели, тёмно-синий текст, крупные скругления, насыщенные синие CTA, общий `SarmaExpressHeader`.

## Стек

- Next.js App Router
- React
- TypeScript
- Tailwind
- Shared package для общих схем и типов
- Backend package для заказов и интеграций

## Навигация

Файл: `frontend/components/sarma-express-header.tsx`.

Верхнее меню:

- `Калькулятор`
- `Отслеживание`
- `Услуги`
- `Бизнесу`
- `Тарифы`
- `Пункты выдачи`

В `Услуги` есть:

- `Доставка из интернет-магазинов РФ` -> `/superbox?flow=pickup_paid`
- `Отправления в РФ` -> `/superbox?flow=ship_russia`
- `Сборные грузы (LTL)` -> `/ltl`
- `Полная загрузка (FTL)` -> `/ftl`
- `Отмена заказа` -> `/cancel-order`

## SUPERBOX

Основной файл: `frontend/components/superbox-app.tsx`.

Важные flow:

- `pickup_paid` - доставка из интернет-магазинов РФ, анкеты маркетплейсов;
- `pickup_standard` - заказ по ссылке;
- `order_lookup` - отслеживание;
- `ship_russia` - отправления по РФ;
- `tariffs` - тарифы;
- `business` - бизнес;
- `home_delivery` - доставка курьером;
- `cancel_order` - отмена заказа;
- `support` - поддержка.

## Пункты выдачи в маркетплейсах

В marketplace-flow пользователь выбирает один из 8 клиентских пунктов выдачи:

- Донецк, ул. Челюскинцев, 184
- Донецк, ул. Куйбышева, 70/13
- Волноваха, ул. Менделеева, 14
- Макеевка, ул. Островского, 3/18
- Горловка, пр. Победы, 16
- Горловка, ул. Интернациональная, 76
- Мелитополь, ул. Горького, 55
- Мариуполь, 60 лет СССР, 8

После выбора показывается адрес в Ростове-на-Дону, куда нужно оформить заказ в выбранном маркетплейсе. Для WB и Ozon адрес зависит от выбранного пункта; для остальных используется фиксированный адрес/терминал из соответствующего листа Excel.

## Заказ по ссылке

`pickup_standard` доступен только для:

- Wildberries
- Ozon
- Яндекс Маркет

Форма принимает имя, фамилию, телефон, размер, ссылку на товар, пункт выдачи и дополнительную информацию. Отправка требует согласия с условиями.

## Отмена заказа

Страница `/cancel-order` рендерит `SuperboxApp` с начальным flow `cancel_order`.

Пользователь вводит номер заказа, подтверждает отмену, backend переводит заказ в статус `CANCELLED`.

## Backend и схемы

Ключевые файлы:

- `shared/src/index.ts` - Zod-схемы и типы заказов;
- `frontend/lib/api.ts` - клиентский API;
- `backend/src/routes/orders.ts` - маршруты заказов;
- `backend/src/services/order-service.ts` - хранение заказов;
- `backend/src/services/bitrix-service.ts` - Bitrix payload.

При добавлении новых полей формы нужно обновлять весь путь: UI -> `frontend/lib/api.ts` -> shared schema -> backend route -> storage/Bitrix.

## Проверки и сборка

Основные команды:

- `npm run build -w shared`
- `npm run build -w backend`
- `npm run build -w frontend`

Для UI-проверок использовался локальный сервер `http://localhost:3000` и Playwright/Chrome.
